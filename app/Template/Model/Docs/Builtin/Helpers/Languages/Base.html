\Tipui\Builtin\Helpers\Languages\Base

Set the base path of translation file to be loaded.
Returns the self instance.

Path precedence

If the file calling the method colides with existing folder under same base, the folder prevails to mount the path base.
ie: app//foo/bar.html calling for Languages::Base(), the path base to load the translation file will be app//foo/, however, if exists the path app//foo/bar/, the folder name bar prevails, mounting the path as app//foo/bar/.

Without parameters

The current script file base as path base reference to load translation file.
ie: In a file in app//foo/bar.html, the path base to load the translation file will be app//foo/

Changing the base

Setting the base to 1 directory above the current path.
ie: For app//foo/bar.html, the current path base is app//foo/.
For more samples, see Languages::Label().
*Cautions!
- Do not use relative paths as ../folder/, that may cause unnexpected errors because the library do not provide interpretation for relative paths.
- The base path must be inside the app folders, otherwise, will throw fatal errors.
